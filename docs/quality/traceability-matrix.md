## Опис тест-кейсів

### TC-01 — Реєстрація користувача
- **Передумова:** сервіс запущено, БД доступна
- **Кроки:** POST `/api/auth/register` з валідними даними
- **Очікуваний результат:** HTTP 201, повернуто `userId`

### TC-02 — Автентифікація
- **Передумова:** користувач існує в системі
- **Кроки:** POST `/api/auth/login` з email та password
- **Очікуваний результат:** HTTP 200, повернуто JWT-токен

### TC-03 — Пошук квитків
- **Передумова:** є доступні квитки в БД
- **Кроки:** GET `/api/tickets?from=Kyiv&to=Lviv&date=2025-06-01`
- **Очікуваний результат:** HTTP 200, список квитків у відповіді

### TC-04 — Створення бронювання
- **Передумова:** користувач авторизований, квиток доступний
- **Кроки:** POST `/api/bookings` з `ticketId`
- **Очікуваний результат:** HTTP 201, статус `pending`

### TC-05 — Оплата бронювання
- **Передумова:** існує бронювання зі статусом `pending`
- **Кроки:** POST `/api/payments` з `bookingId` та платіжними даними
- **Очікуваний результат:** HTTP 200, статус бронювання змінено на `confirmed`

### TC-06 — Скасування замовлення
- **Передумова:** існує активне бронювання
- **Кроки:** DELETE `/api/bookings/{id}`
- **Очікуваний результат:** HTTP 200, статус `cancelled`

### TC-07 — Перегляд історії замовлень
- **Передумова:** користувач авторизований, має замовлення
- **Кроки:** GET `/api/bookings`
- **Очікуваний результат:** HTTP 200, список замовлень користувача
