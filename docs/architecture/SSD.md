# SSD — System Specification Document

> **Версія:** v1.0 | **Дата:** 2025-04-01  
> **Статус:** Затверджено  
> **Джерело:** Це головний документ вимог (Single Source of Truth)

---

## 1. Призначення системи

REST API сервіс для бронювання квитків призначений для обробки запитів клієнтів щодо пошуку, бронювання та оплати квитків.

Система надає можливість:
- створення замовлень;
- перегляду доступних рейсів або подій;
- керування обліковими записами користувачів.

**Формат взаємодії:** виключно API (JSON через HTTP).

## 3. Функціональні вимоги

| ID | Функція | Опис |
|----|---------|------|
| FR-01 | Реєстрація користувача | Новий користувач може зареєструватися в системі |
| FR-02 | Автентифікація | Користувач входить через JWT-токен |
| FR-03 | Пошук квитків | Пошук за маршрутом, датою, категорією |
| FR-04 | Створення бронювання | Клієнт може забронювати квиток |
| FR-05 | Оплата бронювання | Оплата через зовнішній платіжний шлюз |
| FR-06 | Скасування замовлення | Клієнт може скасувати активне бронювання |
| FR-07 | Перегляд історії | Клієнт бачить список своїх замовлень |

---

## 4. Нефункціональні вимоги

- **Продуктивність:** не менше 500 запитів/хв
- **Час відповіді:** до 300 мс
- **Доступність:** 99.9% uptime
- **Безпека:** HTTPS, JWT авторизація
- **Масштабованість:** горизонтальне масштабування

---

## 5. Обмеження та припущення

- Система працює виключно через REST API
- Використовується одна основна реляційна база даних (PostgreSQL)
- Платежі обробляються зовнішнім сервісом
- Очікується високе навантаження в пікові години
